# PRD - Product Requirements Document
## Making a Todo App

## 1- Objective
Build a HTML website for Todo App that allows users to create manage and track task easily.

## 2- Scope
- Task mgmt like - add, edit, delete, task completed
- responsive design for mobile and desktop

## 3- Features and Requirements

### P1 ( must have features)
- Add new tasks with title and description
- Edit and delete tasks
- mark tasks as complete / incomplete
- categorize tasks ( work, personal, custom )
- set due dates
- Responsive UI

### P2 ( Good to have features)
- search and filter task
- Notificaitons and reminders for due date.
- Dark Mode 

## 4- User Stories or Use Cases ( purpose of my software why software will be used by the user )
- As a user, I want to quickly add tasks so I dont forget them
- As a user, I want to mark task complete so I can track progress
- As a user, I want to categorize tasks so I can seperate work and personal life
- As a user, I want reminders so I dont miss deadlines.

## 5- Technical Requirements
- HTML, CSS , Javascript
- Git + Github for version control
- Deployment on netlify/vercel

## 6- Design Requirements
- Clean, Minimal UI
- Responsive layout
- Consistent color Scheme 

## 7- Success Metrics
- Number of active users per month
- Average task created per user
- Task Completion rate.
- User Retention after 30 days.
- Positive feedback on usability

## 8- Timeline
- 3 months / 6 months / 12 months
