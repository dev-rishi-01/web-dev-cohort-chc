# Assignment – Chai Cohort

- Onboarding Documentation Requirement  
- This document will serve as onboarding material for any new employee or intern joining Chai Cohort. It should guide them on how to properly use and maintain the company’s Git repository.  

- Git Repository Name: Chai Cohort  

## Requirements  

### Basic Git Commands  
Include all essential Git commands and explain how to use them. The document should clarify how Git works and how team members should interact with the repository.  

### Collaboration Guidelines  
When collaborating on the repository, employees should keep certain important practices in mind. These guidelines should be clearly outlined so team members can work smoothly together without conflicts.  

### Commit Rules  
Specify clear rules for writing commit messages:  
- How commits should be structured  
- How commit messages should be written  
- What information must be included in each commit message  

### Issue or Ticket Resolution  
When resolving any task, issue, or ticket, contributors must include the ticket ID in their commit messages to maintain traceability.  

### Company Rules  
These standards and best practices should reflect the official Git usage policies of Chai Cohort, as this will be the company’s formal documentation.  

### Screenshots Required  
Add relevant screenshots throughout the document to help new team members visually understand how to use Git and follow the workflows properly.  

---

## Final Output Format  

- The final onboarding document should be written in the style of a 2–3 page blog.  
- It must be clear, beginner-friendly, detailed, and visually supported with screenshots.
